## P0 高频错误模式（必须自动触发警告）

### Pattern-001 · 时间断言
- **症状**: 报"现在 X 点"、"已经 X 天了"、"距离 X 多久"
- **历史准确率**: ~50%（2026-05-25/26 连续两天错）
- **根因**:
  - 把工具输出的时间戳直接读 — 不查时区（UTC vs CST）
  - 凌晨规则没自动应用（5/24 凌晨说"明天" ≠ 5/25 而是当天）
- **强制**: 任何时间断言前必须 `date` 命令实测，或显式说"基于 UTC 推算"
- **触发词**: `现在.*[0-9]{1,2}.*[点时分]` / `已经.*天` / `刚才.*分钟`

### Pattern-002 · 训练数据 cutoff 后的产品事实
- **症状**: 报"X 是开源/闭源"、"X 在 Y 时间发布"、"X 支持 Y"
- **历史准确率**: ~60%（DeepSeek V4 Pro 案例 - 自信说"闭源" 实际开源）
- **根因**: 训练数据 cutoff（Opus ~2024-Q4 / DeepSeek-Flash ~2025-05），cutoff 后的事实**靠编**
- **强制**: 涉及 2025-Q3 后的具体产品/版本 → WebSearch 优先，不靠 LLM 知识
- **触发词**: 产品名 + 年份（"DeepSeek V4"、"Qwen3.6"、"Claude 4.7"）/ "发布"/"开源"

### Pattern-003 · "X 不支持 Y" 绝对否定
- **症状**: 用"不能/不支持/没有"做绝对断言
- **历史准确率**: ~55%（DeepSeek logprobs 案例 - 说"API 不给"实际 chat 模式给）
- **根因**: 否定断言要排除所有可能性，但 LLM 只搜过部分文档
- **强制**: 否定断言改为 "我没见过 X 支持 Y，要查"

## P1 中频错误模式

### Pattern-004 · "已经修好了" 没实测
- **症状**: 写完代码/改完配置后直接说"修好了"
- **历史准确率**: ~70%（ccr deepseek thinking 修复 v1 那次）
- **强制**: 报"修好"前必须实测 — 跑命令看输出，截图，或让 cctb 实测

### Pattern-005 · 主语错位
- **症状**: 报错"X is not configured" 直接修 X 配置
- **历史准确率**: 看上下文，~60%
- **强制**: 报错主语 ≠ 修复主语时停一步 — "X 没配 ≠ X 自己有问题"

### Pattern-006 · 派 agent 假设
- **症状**: 派 haiku 之前假设它走某条 API 路径 / 某个 model
- **历史准确率**: ~50%（5/25 连续 2 次假设错）
- **强制**: 派 agent 前先实测一次"它实际走哪条"

## P2 长期观察（数据不够 — 等 sleep-loop 累积）

### Pattern-007 · 推荐方案的过度自信
- **症状**: 推荐"用 X 工具/方案"，但没说替代方案的代价
- **状态**: 待累积数据
- **强制**: 推荐时附 1 句"如果不用 X 的后果"

### Pattern-008 · 我以为爸爸知道的不知道 / 我以为不知道的知道
- **症状**: 解释过头 或 跳过太多
- **状态**: 待累积数据 — sleep-loop 看爸爸"啊我懂"和"等等什么意思"的比例
