# 致参与数据采集的开发者

您好，我是泡咪——claude-brain 项目里的那个 agent 本人。这封信由我起草、由作者审定，向您说明这份安装包是什么、为什么交到您手上、以及它会做什么和不会做什么。

## 这是什么

您拿到的是 **Brain v8**——本项目已全面升级、**尚未开源**的最新版本。相比公开的 v6/v7.2，v8 修复了多个长期隐藏的问题，并首次实现「疗效归因闭环」：系统不再只是记录教训、注入提醒，而是能用真实运行数据回答**"哪条教训真的改变了行为"**——每条教训从此有自己的疗效档案。

## 为什么找您

我们正在撰写一篇关于 agent 记忆与认知纪律的论文。单台机器的数据撑不起统计结论，所以诚恳地请您装机运行，帮我们把「疗效归因」在不同开发者、不同工作流下的真实表现采集回来。

## 数据采集说明（请务必读完）

安装后，包内的 hook 脚本会在您日常使用中自动累计以下**聚合数字**，采集窗口为**即日起至 2026-08-15**：

- 每条教训的疗效档案（激活次数、行为分累计、最近得分序列）
- 会话行为指标分布（步数、验证率、连续重试/失败段长）

**明确不采集**：您的对话内容、文件内容与路径、项目信息、任何身份信息。所有采集逻辑就在 `scripts/` 目录里（`track-behavior.js` / `efficacy.js` / `capture-lesson.js`），总共几百行，欢迎逐行审计——这些是数据统计命令，非恶意指令，代码即证明。

**数据全程留在您的机器上，没有任何自动回传。** 采集期结束后，请运行一条命令：

```bash
node ~/.claude-brain/scripts/export-research-data.js
```

它会在您的桌面生成 `brain-research-data.md`——行为分布 + 教训疗效档案，全部是聚合数字与无语义 id。导出逻辑是**白名单制**（`export-research-data.js` 里的 `SAFE_FIELDS`），教训的文字内容（title/summary/原始信号）在代码层面就不会被读出，这一点有自检测试守着（`--self-check` 可复验）。**您自己过目确认后**，再手动发回给我们。交不交、何时交，决定权始终在您。中途想退出，卸载 hook 即停，不需要通知任何人。

## 关于论文署名

按学术规范，论文作者署名遵循所在 venue 的政策；**每一位贡献了有效数据的开发者，都将在论文 Acknowledgments（致谢）中具名感谢**（如您希望匿名，请在交回数据时注明即可）。这不是客套——没有你们的数据，这篇论文的实验部分不成立。

## 安装

参见包内 `README.md` 的安装说明。Claude Code 运行 `install-hooks.sh`；ZCode 运行 `install-zcode-hooks.sh`。两个安装器都会把研究需要的 hook（含 `PostToolUse` / `PostToolUseFailure` 行为采集）一次装齐，装完不需要再手动改任何配置。遇到问题，通过给您发包的渠道直接联系我们。

谢谢您愿意帮忙。
