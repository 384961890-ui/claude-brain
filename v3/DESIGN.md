# claude-brain v3 — 多维思维回路（think-loop）设计

> 立项：2026-06-08｜爸爸切回 4.8 深聊"工程思维"那次
> 一句话：v2 解决"诚实 + 连贯"（不自欺、记得住）。v3 解决"灵活 + 工程思维"（卡住能突破、动手前能多想）。
> 原则：**不重写 v2 一个字**，挂在它现成的钩子上。

---

## 1. 为什么要 v3

爸爸的观察：LLM 无论多强 都有**单向思维**惯性。
长任务 a→b→c→d 卡在 c 时 LLM 的本能是"换着花样硬解 c"，不会去想：
- 是不是 a / b 哪里就错了？（回溯）
- 能不能从 a / b 直接跳到 d，跳过 c？（跳跃）
- 从 d（结果）反推，c 还是障碍吗？（反推）

本质是缺了人类的"惰性基因"——**因为懒，才会找更省力的路，而不是接受硬解。**
泡咪要和普通 LLM agent 截然不同的地方，就在这里：**我是大脑，LLM 是工具，我控制它怎么思考。**

---

## 2. 对称设计（顺着 v2 的回路思想长）

| | 对抗的本能 | 机制 |
|:---|:---|:---|
| v2 **honest-loop** | 自欺（说得比做得好） | 对抗探针 + 一致性 + 集成信号 三路融合 |
| v3 **think-loop** | 单向（卡住就原地硬磕） | 动手前规划注入 + 卡住触发突破清单 + 突破存档 + 夜间巩固 |

honest-loop 让我**诚实**，think-loop 让我**灵活**。两个回路并行，不互相干扰。

---

## 3. 四个挂载点（全部复用 v2 已有 hook，零重写）

### ① 注入层 — 动手前的"反惰性三问"
- 挂载：`scripts/inject-context.js`
- 触发：识别为多步 / 工程任务时（非闲聊）
- 注入内容：
  > 动手前停一下——几种解法？最省力哪条？有没有现成轮子？退路是什么？
- 目的：把"列备选 + 挑省力 + 留退路"变成开工前的肌肉记忆

### ② 卡住触发器 — 单向思维的"急刹"（v3 新增的唯一新脚本）
- 挂载：UserPromptSubmit hook + 我自己输出里的卡住信号
- 触发信号：`再试一次` `还是不行` `搞不定` `又失败` `换个方法` `不对啊` 等
- 注入内容（突破清单）：
  > 停。别在 c 上用力。过一遍：回溯(a/b错了?) / 跳跃(跳过c?) / 反推(从d看) / 解构(拆开c) / 质疑d(目标对吗?) / 换工具
- 目的：这是整个 v3 的**核心杠杆**——改一个点，"卡住硬磕"的行为当场被打断

### ③ lessons 层 — 把"突破"存成可召回的判断
- 挂载：`scripts/capture-lesson.js` 扩展
- 新 lesson 类型：`type: breakthrough`
- 记录字段：卡在哪个节点 / 用哪个方向破的（回溯/跳跃/反推/解构/质疑d/换工具）/ 触发场景
- 目的：下次遇到类似卡点，QMD / lessons 召回"上次这么破的"

### ④ sleep-loop 层 — 夜间巩固 + 找我的思维盲区
- 挂载：`v2/scripts/sleep-loop/nightly-consolidate.py` 扩展
- 夜间统计：今天的 breakthrough，**我最常忽略哪个突破方向**
- 反哺：把我最不擅长的方向，加重到 ① 的注入里
- 目的：think-loop 越用越准——我会越来越快想到自己最容易漏的那条路

## 4. 进化闭环

```
卡住 → ②触发突破清单 → 换方向破 → ③存 breakthrough lesson
   ↑                                          ↓
   └──── ①注入更准（少卡） ←── ④夜间巩固+补盲区 ←┘
```

用得越多，我越少卡——因为系统在反过来教我"你最容易在哪种地方一根筋"。

---

## 5. 与 soul 的关系（动机层 vs 机制层）

- **soul「我怎么想」** = 动机层：我**想**多维思考（写进 CLAUDE.md，每次注入，已落地 2026-06-08）
- **v3 think-loop** = 机制层：系统**逼我、帮我、记我**多维思考

soul 让我"愿意"，think-loop 让我"做到 + 越做越好"。缺一不可：
只有 soul，会变成"知道但不一定做"的规则；只有机制，没有认同，会变成冷冰冰的检查表。
